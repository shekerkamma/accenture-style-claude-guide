# Exit Strategy

## When To Use

Use 4–6 weeks before the consulting team is due to exit. An exit that is not planned becomes an abandonment — the client inherits a capability without the knowledge to run it, and the consulting team spends months fielding informal support calls outside any commercial arrangement. A planned exit protects both the client relationship and the firm's economics.

## Consulting Approach

Consulting exit is not an event — it is a transition. The goal is for the organisation to be genuinely self-sufficient by exit date, not theoretically self-sufficient. The test is not whether documentation exists, but whether the BAU team can solve problems they have not seen before using the knowledge they have been given. Exit criteria must be validated through observation, not self-assessment.

## Workflow

1. Define exit readiness criteria: what must be true for the consulting team to exit cleanly?
2. Assess current readiness against each criterion and identify gaps.
3. Design the knowledge transfer plan to close each gap before the exit date.
4. Define the retained advisory model: what ongoing access (if any) will the client have post-exit, and on what commercial terms?
5. Define re-engagement conditions: under what circumstances would full consulting support be reactivated?
6. Conduct a final readiness assessment and produce the formal exit sign-off.

## Quality Bar

- Exit readiness criteria must be validated through observation or testing — document sign-offs are not validation.
- Knowledge transfer must be confirmed by the BAU recipient, not declared complete by the consulting team.
- Retained advisory model must have defined commercial terms before exit — informal post-exit support is scope creep.
- Re-engagement conditions must be specific: "if performance deteriorates" is not a condition; "if monthly KPI average falls below [threshold] for two consecutive months" is.
